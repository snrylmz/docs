Knowledge quality is the strongest driver of chatbot answer quality.

## Sections

- `Files`
- `Texts`
- `QA`

## Document lifecycle

1. Create/upload
2. Wait for indexing
3. Activate (`is_active=true`)
4. Monitor quality
5. Update/deactivate as needed

## Files tab

### Supported formats

- `.pdf`
- `.doc`
- `.docx`
- `.txt`

### Upload methods

- Drag and drop
- File picker

### Live limits panel

Shows:

- Document count usage
- Total size usage
- Per-file max size

### Common errors

- Per-file size exceeded
- Total size exceeded
- Max document count exceeded

## Texts tab

Best for fast operational content:

- Procedures
- Policy updates
- Campaign details

Steps:

1. Set title
2. Write content
3. Save

## QA tab

Best for deterministic question-answer pairs:

- FAQ answers
- Delivery/returns/pricing clarity

Steps:

1. Title
2. Question
3. Answer
4. Save

## List and statuses

Document table shows:

- Name
- Size
- Status (`active` / `draft`)
- Index state (`indexed`, `chunks_count`)
- Created date

## Bulk actions

For selected items:

- Bulk delete
- Bulk activate
- Bulk deactivate
- Single preview

## Preview usage

Use preview to verify:

- Correct document selected
- Correct extracted content
- Correct index state

## Content quality standards

- Split long text with headings
- Merge duplicate topics
- Reduce ambiguous wording
- Deactivate stale content
